Github Notları

Temel Terimler

  1. repository:Kısa ismiyle repo, kodlarımızın saklandığı yerdir.
  2. master:Depoadaki ana proje sürümüdür.
  3. branch:Git ağaç mantığı ile çalışır ve projedeki geliştirmelerimizi ağaç daklı mantığı ile ayrı dallarda saklayabilir, sonrasında master branch da birleştirebiliriz.
  4. merge:Branchlerde yapılan işlemlerin master da birleştirilmesidir.
  5. commit:Commit işlemi lokaldaki dosyalarımızı depoya gönderilmek üzere hazırlar.
  6. puh:Dosyalarımızı depoya göndermemizi sağlar.
  7. pull:Depodaki dosyalarımızı çekmemizi sağlar.
  8. .gittignore:Git'in takip etmesini istediğimiz dosya ve klasörleri belirttiğimiz dosyadır.

VS Code'da prolede bir değişiklik yapıldıktan sonra bunun Github'a göndermek için source control taabını açtık, sonra message alanına yapılan değişiklikle alakalı mesaj yazdık son olarak commit butonuna basıp depoya gönderilmek üzere hazırladık ve sync butonuna basarak puh işlemi yaparak değişikliği Github'a yolladık.

Temel Git Komutları

  1. git add veya git add*, git add index.html:Klasöre eklenen dosyaları gönderilmek üzere sahneye alma(stage)
  2. git status:Klasördeki tüm değişiklikleri göster
  3. git commit - m "İlk commit":Depoya gönderilmek için commit işlemi yapılır.
  4. git commit -a a "Mesajımız":Stanging'e ele ve commitle
  5. git push:Değişiklikleri depoya gönderme komutu
  6. git pull:Uzak depodaki dosyaları lokale indirme
